# 00 — 文档索引与开工入口

最后更新：2026-07-19

## 每次开工的入口顺序

1. 先读仓库根目录 [`AGENTS.md`](../AGENTS.md)；
2. 再读本索引；
3. 再读 [`01-当前任务.md`](01-当前任务.md)；
4. 根据当前任务读取对应的正式设计和权威来源；
5. MALF/Data 任务必须核验 [`02-权威来源登记.md`](02-权威来源登记.md) 中的绝对路径与 SHA-256。

聊天记录、`.superpowers/brainstorm` 和旧仓库均不得替代此入口链。

- [`01-当前任务.md`](01-当前任务.md) — 当前阶段、允许动作、停止条件和下一门禁。

- [`03-设计批准记录.md`](03-设计批准记录.md) — 用户批准的设计第 1–6 部分及替代关系。

## 状态标签

- `implemented`：已经存在实现并通过对应验收；
- `verified`：合成语义链路通过完整追踪验证；
- `research_only`：可用于历史研究，不得表示当前或 operational 状态；
- `design-only`：只有已批准设计，没有实现；
- `not-started`：尚未开始；
- `rejected`：未通过硬门禁。

## 冲突处理

如果文档之间冲突：

1. 停止业务施工；
2. 保留冲突证据；
3. 按 `AGENTS.md` 的权威链裁决；
4. 通过新增决策记录说明 supersedes 关系；
5. 禁止静默覆盖历史文档。
