## `RB-CLARIFY-001`——ETF fixed-point adapter 精度合同澄清

状态：**recorded**。

登记日期：`2026-07-20`。

本记录不改变设计第 3 部分已经批准的 `raw_none` 原始整数严格比较，也不新增业务功能；它只裁决曾混淆的术语和一致性边界：

1. 撤回“RiskBench v0.1 缺少交易所 `tick_size` 字段或 tick-size 对齐算法”的问题表述。v0.1 不以 exchange `tick_size` 为运行字段，也不进行 tick-size quantize；
2. 保留并强化已批准的原始整数 fixed-point 合同，正式比较域为 `source_integer_fixed_point`，版本为 `source_integer_fixed_point-v0.1`；
3. `malf-v2.0-etf-tick-v0.1` 是 RiskBench 的 ETF adapter variant 标识。它不修改 Definitive 原文，但也不得宣称为 Definitive Core O3 `round(2)` 归一化的等价实现；
4. 该 variant 只适用于 `raw_none / research_only`；其规则版本与 `source_integer_fixed_point-v0.1` 必须进入已发布快照的 `rule_versions`；
5. `qfq_back` 不继承本合同，必须另行定义精度、fixture、规则版本、测试和审批。

对 `RB-M03-T01` 的影响：实施前必须将上述非等价性、禁止 tick-size runtime 处理和发布谱系要求写入任务实现计划与 V3 证据；本澄清不授权安装依赖、业务施工或该任务提前开始。
